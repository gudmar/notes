Binding data in angluar
Binding is allowing automatic sync between data in component and view. There can be one direction data flow, or two direction data binding. In angular there are:
  • property binding
  • interpolation: one way variable to text in html binding
  • Attribute binding: in case there is no HTML propery like in aria case <button [attr.aria-label]="closeLabel">
  • Event binding
  • Two way data binding: the same as property binding + event binding used together.

Interpolation

let someStringVariable: string = 'someString';
<p>This is someString{{someStringVariable}}</p>
Now every time someStringVariable changes in component, view will be updated. But in case p element in view changes, someStringVariable in component will remain unchanged.

Property binding

Binds a property in component to property in HTML. Used like this [binding-target]='binding-source'. Binding source can be a template expression, property in component, method in component or expression containing all of them. Style and class binding is a property binding, but for more information see chapter related to directives. 
        <h1 [innerText] = "title"></h1>
        <button [disabled] = 'isDisabled'>X
In above example title variable is bound to innerText property, so it will show up as content of h1.

Event binding

Is a one way binding from view to component, so opposite direction as property or interpolation. <button (click)="onSave()">Save</button>. This example ilustrates how event binding should be used.

Two way biniding

This is a commination of property binding and event binding. Often a ngModel directive is used for this binding. ngModelChange is a angular specyfic event, that is an output of ngModel, and is triggered every time an event occures. 
<input type = "text" [value] = "name" (input)="name=&event.target.value"
equals to 
<input type = [(value)]='name'>
Note: if Property ‘value’ does not exist on type ‘EventTarget’ occures, it means that there is no guarantee, that velue property exists on input element. In this case:
  • Cast can be used ($any($event.target).value)
  • fullTemplateCheck in tsconfing.json can be set to true
NOTE: If this banana in the box syntax is used [()], then there needs to be a valueChange property under element. eg [(somePropery)] = 'vlaue', there needs to be a someProperyChange property under elemnt. If there is no such property, this will not work. In such case ngModel comes handy. 
    import { FormsModule } from '@angular/forms';

    <input type="text" name="value" [(ngModel)]="value">
    <p> You entered {{value}}</p>
    <button (click)="clearValue()">Clear</button>
This is important, that ngModel is in @angular/forms, not angular core library, so it has to be imported.

ngModelChange

In below case there will be 2 events triggered when input is changed in order of occurance. ngModelChange is triggered every time input changes, and JS change event will be triggered every time input loses focus. 
    <input [ngModel]="firstName (ngModelChange)="firstName = $event" (ngModelChange)="firstNameChanged($event)"/>
Services and dependency injection
Modules
A module can be compared to a folder, or namespace. It is a way to divide a complex application into snaller pieces for readibility. There needs to be at least one module, as bootstrapping - a proces of loading NG application, is related to loading a root module. Then there are more modules. Most commonly each 'page' should be a module. This packages are sometimes called features. If a page is too big, it needs to be devided into submodules.
Angular itself consists of modules such as:
  • ApplicationModule from @angular/core library configurating the root injector for app.
  • BrowserModule, BrowserTransferStateModule, HammerModule from @angular/plafgorm-browser library. All are related to angular support for different browsers
  • https://angular.io/api/platform-server">ServerModule and ServerTransferModule that is a ng module for server
  • FormsModule from @angular/forms library, for making template-driven forms
  • RouterModule from @angular/router for lazy page loading using async.
Module consolidate components, diractives and pipes into functional blocks. @NgModule is a decorator taking:
  • declarations[]: an array of pipes, directives and components that are part of current module, and that are needed to be available to other components, directives and pipes of current component. If something is not included in this array, then it will be not visible in current module
  • imports[]: is an array of other modules that will be used inside current module.
  • providers[]: is an array of services that will be used in this module. It needs to be treated separately, because dependencies are injected
  • bootstrap[]: array of components to be loaded while component is loading
Components
A component is a piece of web application havnig its own view, logic and styling. In angluar component has a HTML template, styling file and a class written in TypeScript. Some components have their services: additional classes with reusable logical services. For example communication with server shoud be done with service, as this feature would be used in other classes and it putting it into class with comonent would deny single responsibility principle. A component is a js module, so it is a separate file importing angular libraries, exporting a class and is decorated with a @Component function taking: selector, template, styling and dependencies as arguments.
  • templateURL: path to HTML template file.
  • styoeURLs: array of styoe sheets.
  • selector: tells angular where to put a component. Is a css selector angular will look for.
  • template: in stead for templateURL, is a string HTML template>
            import { Component } from '@angular/core';

            @Component({
                selector: 'app',                    // HTML  tag, where component will be injected
                template: '<h1>{{title}}<',   // this can be substituted with templateURI
                style: 'h1{color: blue;}'           // styleURI
            })

Directives
A directive is a sort of angular specific attribute, that is a part of HTML template. Its task is to manipulate DOM (change appeariance or behavior, or even add HTML elemens)
  • Component directives
    • Directives containing a template (have their own view)
  • Structural directives
    • Can change DOM layout by adding or removing elements.
    • Start with an asterix *
    • *ngFor can add elements using list that is defined in TS component. 
          <tr *ngFor="let car of cars">
        <td>{{car.nrOfSeets}}</td>
        <td>{{car.horsePower}}</td>
        <td>{{car.has4WD}}</td>
    </tr>
      This will put a row in table for every car in car list.
    • *ngSwitch choses one or more components depending on variable content. Bounded to *ngSwitchCase and *ngSwitchDefault
    • *ngIf Html element having this directive renders if condition is met
  • Attribute directives

*ngSwitch

  • Structural directive
  • Bound to expression, so if expression somewhere in class will change elements will be rerendered
  • *ngSwitchCase is should be used only in element tested in ngSwitch element
  • Each element with ngSwitchCase matching expression will be added to DOM, not only first one
  • ngSwitchDefault element is rendered only if none of ngSwitchCase elements are matched
  • There can be more than one ngSwitchDefault elements. If nonoe ngSwitchCase is matched tan all ngSwitchDefault will be added
  • ngSwitchDefault can be anywhere in ngSwitch block, does not have to be at the bottom
  • There can be elements inside ngSwitch statement without any of ngSwitchCase or ngSwitchDefault directives. In such case they are displayed normally.
  • Empty string as an expression is 0
  • ngTemplateOutlet is a directive that places ng-template instead of ng-container having a ngTemplateOutlet directive (example below)
        <div [ngSwitch]="someExpression">
            <div *ngSwitchCase="matchExpression1">someTemplate</div>
            <div *ngSwitchCase="matchExpression2">someOtherTemplate</div>
            <div *ngSwitchDefault="lastExpression">finalExpression</div>
        </div>

*ngIf

  • Renders only if condition is true
  • Structural directive
  • ngIf != [hidden] = 'condition', because ngIf removes element from dom, it does not hide it.
  • Is boud to condition variable, so if variable changes from false to true elent should appear, and if statement changes to false then element will be removed.
  • else statemet can be simulated with negation *ngIf = "!condition", but cleaner way is to use 'else' block like in example below.
    <div *ngIf="someCondition">
        This div will be rendered only if someCondition is met
    <div>
else block 
    <div *ngIf="condition; else elseBlock">
        content to render, when the condition is true 
    </div>
        
    <ng-template #elseBlock>
        content to render, when the condition is false 
    </ng-template>}}
ng-template can be used anywhere in code, but good to place after if directive for readibility. Will not render if condition is false. There is a possibility to use if then else statement: 
    <p *ngIf="showMe; then thenBlock2 else elseBlock2">
        This is not rendered
    </p>
    
    <ng-template #thenBlock2>
        <p>ShowMe is checked Using thenblock

    </ng-template>
    
    <ng-template #elseBlock2>
        <p>ShowMe is unchecked Using elseBlock

    </ng-template>

ngClass

Used to add or remove classes from element.
  • ... this will evaluate expression (in this case 'cssClass1 and cssClass2' so not too much to evaluate), and add classes from evaluated expression.
  • ... This adds classes to element without evaluation
  • ... This does the same as first example, binds expression to calss, but with array.
  • ... This adds classes with true value, and removes classes with false value
  • let someString = 'class1 class2', <element [ngClass]=someString> This will bind a string to class, and each time string changes classes in DOM will be updated
  • let someArray = ['class1', 'class2'], <element [ngClass]=someArray> This will bind someArray to element class attribute, and each time someArray is changed class attribute is updated
  • class CssClass {red: boolean= true; size20: boolean: true;}; cssClass: CssClass = new CssClass(); <div [ngClass]=cssClass In this case if classes in cssClass change class attribute in DOM will also change.

ngStyle

Modifies elements style
  • <element [ngStyle]="{'font-size': '20px'} binds stlye to expression, and that expression in this case is constatn, so noting will be reevaluated
  • let size = '20px'; <element [ngStyle]="{font-size': size} in this case font size of element is bound to size variable, so it will change as often as size variable changes.
  • let size = 20; <element [ngStlye]="{'font-size.px': size}" this binds font size of element to size variable. Here size is a number and size will be in pixels.
  • Style can be bound to a controller class, and every time styling in that class chages, element styling will also change (see below)
class StyleClass {
    'color': string = 'red';
    'font-size.rem': number: 1.2;
}
someStyling: StyleClass = new StyleClass();
<div [ngStyle] = "someStyling">
Every time someting in someStyling changes, element style will also change.

*ngFor

This will iterate through a TS list and insert element to DOM for every elemetn in the list. However every time something little changes in this list whole list will have to be rerendered. To manipulate elements in DOM that changed, and leave rest there should be a sort of unique ID to track those elements (like in react). For this there is a trackedBy keyword 
trackByFn(index, item) {
    return item.title;
}

<ul>
    <li *ngFor="let movie of movies; let i=index;trackBy: trackByFn;">
        {{i}}. {{ movie.title }} - {{movie.director}} <button (click)="remove(i)">remove</button>
    </li>
</ul>

Using parent chid communication

@input() decorator (import {Input} from @angular/core) can be used for communication from parent to chid:

in chid component

import { Component, Input } from '@angular/core'; // First, import Input
export class ItemDetailComponent {
    @Input() item = ''; // decorate the property with @Input()
}
chid components template is 
<p>
    Today's item: {{item}}
</p>

parent element will have:

export class AppComponent {
    currentItem = 'Television';
}
and its template whould include child component with a property binding: 
    <app-item-detail [item]="currentItem"></app-item-detail>

@output() decorator (imprt {Input, EventEmitter} from @angular/core) can be used for comunication chid to parent with help of event.

Child component configuration:

import { Output, EventEmitter } from '@angular/core';                        
@Output() newItemEvent = new EventEmitter<string>();

addNewItem(value: string) {
    this.newItemEvent.emit(value);
  }
EventEmitter<string>() tells Angular, to create a new event emitter and that data it emits is of string type. Now template: 
<label for="item-input">Add an item:</label>
<input type="text" id="item-input" #newItem>
<button (click)="addNewItem(newItem.value)">Add to parent's list</button>
#newItem is a template reference vairable. So on click event will be triggered and value passed with this event will be taken from input named #newItem

Parent configuration

export class AppComponent {
    items = ['item1', 'item2', 'item3', 'item4'];
    
    addItem(newItem: string) {
        this.items.push(newItem);
    }
    }
Now a part of parents template. This event binding binds newItemEvent in child to addItem() function of parent. 
    <app-item-output (newItemEvent)="addItem($event)"></app-item-output>
