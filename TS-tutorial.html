About
Java Script is a world standard interpreter script language for front end developement. It has a variety of methods that can be used for DOM manipulation and communication with servers. This invented in 1996 by Netscape langulage is the only supported by every brawser language. I does not need to be compiled to run. Just needs an interpreter in brawser or installed command line interpreter (Node.js) to run. Howver it comes with some disadvantages. The biggest one, leading to many bugs, is no variable type control. This disadvantage is removed by superset of js: Type Script. This invented in 2012 in Microsoft language needs a compiler (tsc) and is compiled to java script. Because of this everything written in js becomes a valid ts code. However TS always check if types of variables are used consequently. It gives warnings and errors in cases when js would not warn us and run code with potential, hard to find, bugs.

Types in details: Interfaces
Interfacing is sometimes named "duck typing". This is because interfaces don't have a representation in js and are not compiled at all. They role is to enable planning a app structure and check proper type usage. Interfaces can be used to:
  • Prototypes of objects
  • Can be used to describe functions
  • Prototypes of classes - can be later used to implement class
  • Can have optional parameters
  • Can have any number of any type properties
  • Can have readonly properties
  • Interface content can be indexed with strings or numbers to create a sort of dictionary
  • Can be extended by another interface
  • Can extend classes

Interface as object: example

interface SomeInterface{    // Note! Capital camel case
    Label: string;
    Id:    number;
}

Interface describing a function

interface SomeInterface{
    Label:                 string;
    Id:                    number;
    (a:number, b:number):  number
}

Optional parameters

interface SomeInterface{
    Label:        string;
    Id:           number;
    optional1?:   string[];
    optional2?:   string|number;
}
This way if optional1 or optional2 are not present in SomeInterface declaration this is not an error. Interfaces can also have a multiple number of interfaces, which quantity or type is not yet specified. Like an object in NoSQL bases.

Any number of properties in interface:

interface SomeInterface{
    Label:               string;
    Id:                  number;
    optional1?:          string[];
    optional2?:          string|number;
    [propname:  string]: any
}
This creates an array of strings, which each element can hold a value of 'any' type

Readonly properties

Const should be used in case of variables, where readOnly should be used in case of properties. 
interface SomeInterface{
    readonly Label:               string;
    readonly Id:                  number;
}
Readonly properties can be written only once, when declared for the first time. Attempt to write something to this property for the second time would give an error during compilation.
There are also readonly arrays, that can be members of interfaces. See chapter about arrays and type variables (type assertion)

Indexing interfaces with strings or numbers:

Interface content can be indexed with numbers: 
interface SomeInterface{
[index: number]:     string;
}

let myArr: SomeInterface = ['Bob', 'Fred'];
// now this is valid
let v1: string = myArr[1];
Content can be also indexed with strings: 
interface SomeInterface{
[index: string]:     string;
}

let myArr: SomeInterface = ['Bob', 'Fred'];
// now this is valid
let v1: string = myArr[1];

Types in details: Objects and Type alias
Objects in TS:
  • Must have a type
    • Objects named differently can be of the same type
    • If object a has properties named the same way that object b, and types of properties are the same and properties have the same attributes (like readonly), then objects are concidered of the same type (duck typing: if a bird walks and talks like a duck, then this is a duck)
    • Object type can be defined with object literal 
      let i = function(a: {key: number, key2: string})
    • Object types can be defined with interfaces: 
      interface SomeInterface{
    key1: number,
    kay2: string
}
funciton doSomething(obj: SomeInterface)
    • Object type can be defined as a alias type: 
      type SomeType{
    key1: number,
    kay2: string
}
funciton doSomething(obj: SomeType)
  • Objects in TS can have (similary to interfaces) optional and readonly parameters: 
    type SomeInterface{
readonly key1: number,
kay2?: string
}
funciton doSomething(obj: SomeInterface)

    Fact, that property is readOnly does not mean, that it cannot be modified in any way! It cannot be rewritten, but can be changed by aliasing.
Types in details: Generic objects

Generick object type is a template for an object, that can be used to create a object of any type. In below example a T will be replaced in every possible place with string In this way there is a most generic and most flexible object template. 

interface Box<T> {
    contents: T;
}

let objBox = Box<string>
  • T in this template can be substituted with everything, even with user types.
  • Not only interfaces can be generic, but also type alias or arrays (by default)
    type someType<T> = T | null | T[];

Some properies of decorators:

  • Class inheriting from decorated class will not receive decorator behavior
  • Decorators are composable. If stacked one on top of another they will be evalulated from top to bottom and executed form bottom to top.

Decorator can be used to e.g. rerender a component every time a setter is used. So something like angular functionality without ingerention to code.
